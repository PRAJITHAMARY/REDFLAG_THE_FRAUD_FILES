## 📖 Project Overview

**REDFLAG_THE_FRAUD_FILES** is a SQL-powered fraud detection project that analyzes financial transaction data to identify suspicious activities and fraudulent behavior. The project uses advanced SQL queries to uncover hidden fraud patterns commonly found in digital payment systems and banking environments.

This project demonstrates how structured data analysis and SQL can be used to detect fraud without requiring machine learning models, making it an excellent example of database analytics and financial intelligence.

## 🧠 Fraud Detection Patterns

| Pattern | Description |
|----------|-------------|
| 🚩 Velocity Fraud | Detects users performing unusually high numbers of transactions in a single day |
| 💳 Card Testing | Identifies repeated failed card transactions used to test stolen cards |
| 💰 Refund Abuse | Detects users with excessive refund transactions |
| 🌙 Odd Hour Activity | Finds transactions occurring during unusual hours |
| 🏦 Mule Accounts | Identifies accounts receiving abnormal credit transactions |
| 🤝 Merchant Collusion | Detects merchants dealing with very few unique customers |
| 💵 Round Amount Clustering | Finds suspicious round-value transactions |
| ⚠️ Just Under Threshold | Detects transactions intentionally kept below reporting limits |
| 📈 Velocity Spike | Identifies sudden increases in monthly transaction volume |
| 🌍 Geographic Impossibility | Finds transactions occurring from different cities within a short period |
| ✅ Failed → Success | Detects failed transactions immediately followed by successful ones |
